# 桌面宠物功能拓展包规范

## 为什么
用户希望一次性为桌面宠物应用添加多个维度的功能，以显著提升其互动性、个性化、生产力价值和智能化水平，使其成为一个更完整、更有趣的桌面伴侣。

## 主要变更
- **引入经济系统**: 添加“专注币”，作为完成专注任务的奖励，并用于在商店中消费。
- **个性化商店**: 创建一个商店，允许用户使用“专注币”购买并切换宠物背景主题和音效包。
- **生产力增强**: 新增网站屏蔽功能，帮助用户在专注期间免受干扰。
- **丰富宠物动态**: 为宠物增加基于其“快乐度”的空-闲动画。
- **外部服务集成**: 接入天气API，使宠物能够提供实时的天气信息。
- **社交模拟**: 创建“访客”系统，让用户保存的其他宠物角色可以偶尔出现并进行互动。

## 影响
- **代码影响**:
  - `src/main.js`: 需要进行大量修改，以集成状态管理（专注币、主题、屏蔽列表）、网站屏蔽逻辑、天气API调用和访客系统计时器。
  - `src/control.html` & `src/control.js`: 需要添加“商店”、“网站屏蔽设置”和“天气设置”的用户界面和交互逻辑。
  - `src/pet.html` & `src/pet.js`: 需要实现主题切换、空闲动画、访客显示和音效播放。
  - `src/styles.css`: 需要为新主题和新动画添加样式。
  - `src/preload.js`: 需要暴露更多从渲染进程到主进程的通信接口。
- **用户影响**: 用户将获得一个功能**更**加丰富、高度可定制且更实用的桌面宠物。

## 新增需求

### 需求1: 商店与个性化
系统应提供一个商店，用户可以使用通过专注获得的“专注币”来解锁新的主题和音效。

#### 场景: 购买并应用主题
- **当** 用户完成一个专注轮次
- **那么** 系统应根据专注时长奖励用户相应数量的“专注币”。
- **当** 用户打开控制面板中的“商店”
- **那么** 应能看到可购买的主题列表及其价格，并显示当前的专注币余额。
- **当** 用户点击一个未解锁且专注币足够的主题的“购买”按钮
- **那么** 系统应扣除相应的专注币，解锁该主题，并播放购买成功音效。
- **当** 用户点击一个已解锁的主题的“应用”按钮
- **那么** 宠物的背景应立刻切换到新主题的样式。

### 需求2: 网站屏蔽器
系统应提供一个网站屏蔽功能，在专注期间阻止用户访问预设的网站列表。

#### 场景: 专注期间屏蔽网站
- **当** 用户在控制面板的“生产力工具”部分添加一个域名（如 `youtube.com`）到屏蔽列表
- **那么** 该域名应被保存。
- **当** 用户启动一个专注轮次
- **那么** 系统应开始拦截所有网络请求。
- **当** 用户试图访问 `youtube.com` 或其任何子域名
- **那么** 访问应被阻止，并显示一个提示页面。
- **当** 专注轮次结束或暂停
- **那么** 用户应能重新正常访问被屏蔽的网站。

### 需求3: 丰富的宠物互动
宠物应有基于其“快乐度”的空闲动画，并且在关键操作时播放音效。

#### 场景: 快乐的宠物
- **当** 宠物的“快乐度”高于70，且用户在一段时间内无操作
- **那么** 宠物应随机播放“开心”的空闲动画（如轻轻摇摆、微笑）。
- **当** 宠物的“快乐度”低于30，且用户在一段时间内无操作
- **那么** 宠物应随机播放“失落”的空闲动画（如垂头、叹气）。
- **当** 用户完成一次购买或完成一个任务
- **那么** 系统应播放积极的确认音效。

### 需求4: 天气查询
系统应允许用户设置城市并向宠物查询当地天气。

#### 场景: 询问天气
- **当** 用户在设置中配置了天气API密钥和城市
- **那么** 系统应能连接到天气服务。
- **当** 用户在聊天框中输入“今天天气怎么样？”或类似问题
- **那么** 宠物应能通过AI调用天气API，并以自然语言回答当前的天气状况。

### 需求5: 访客系统
系统应能让用户已保存的其他宠物角色作为“访客”偶尔出现。

#### 场景: 访客出现
- **当** 应用运行时
- **那么** 系统应每隔一段随机时间（例如30-60分钟）检查一次。
- **如果** 用户拥有多个已保存的宠物角色
- **那么** 系统会随机选择一个非当前激活的角色作为“访客”。
- **那么** “访客”宠物会出现在当前宠物的旁边，进行一段简短的预设对话，然后消失。
